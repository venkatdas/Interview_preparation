# React Application Security — 3-Layer Architecture

> Every request a user makes travels through 3 layers. Each layer is a checkpoint.  
> If one layer fails, the next one catches it. Defense in depth.

```
Browser (React)  →  Middle Layer (Proxy/Gateway/CDN)  →  Backend (Node/Express)
     Layer 1                    Layer 2                         Layer 3
  "What the user              "The gatekeeper               "The enforcer —
   sees & sends"               before your server"            nothing passes unchecked"
```

---

# LAYER 1 — Frontend (React / Client)

> The frontend is **untrusted territory**. Anything here can be bypassed by a user opening DevTools.  
> Frontend security = UX protection + reducing attack surface. Real enforcement is always Layer 3.

---

## 1.1 XSS Prevention

### What is it?
Cross-Site Scripting — attacker injects a malicious script that runs in another user's browser, stealing cookies, tokens, or hijacking the session.

### How to secure

**React JSX auto-escapes — use it:**
```jsx
// ✅ Safe — React escapes this automatically
const userInput = '<script>alert("xss")</script>';
return <div>{userInput}</div>;
// Renders as text, not as HTML

// ❌ Bypasses React's protection
return <div dangerouslySetInnerHTML={{ __html: userInput }} />;
```

**If you MUST render HTML — sanitize with DOMPurify:**
```jsx
import DOMPurify from 'dompurify';

function RichTextDisplay({ htmlContent }) {
  const clean = DOMPurify.sanitize(htmlContent, {
    ALLOWED_TAGS: ['b', 'i', 'em', 'strong', 'p', 'br'],
    ALLOWED_ATTR: []   // no attributes at all
  });
  return <div dangerouslySetInnerHTML={{ __html: clean }} />;
}
```

**Validate URLs before rendering in href/src:**
```jsx
// ❌ Attacker can pass: javascript:alert(document.cookie)
<a href={userUrl}>Visit</a>

// ✅ Whitelist safe protocols
function SafeLink({ url, children }) {
  const isValid = /^https?:\/\//i.test(url);
  if (!isValid) return <span>{children}</span>;
  return <a href={url} rel="noopener noreferrer" target="_blank">{children}</a>;
}
```

## 1.2 Client-Side Input Validation

### What is it?
Validating form data before it's even sent to the server — gives fast UX feedback and reduces bad traffic.

> ⚠️ This is UX, not security. Always re-validate on the backend. Users can bypass frontend validation entirely via DevTools or Postman.

### How to secure

**Zod + React Hook Form (production pattern):**
```jsx
import { z } from 'zod';
import { useForm } from 'react-hook-form';
import { zodResolver } from '@hookform/resolvers/zod';

const loginSchema = z.object({
  email: z.string().email('Enter a valid email'),
  password: z
    .string()
    .min(8, 'Min 8 characters')
    .max(64, 'Max 64 characters')
    .regex(/[A-Z]/, 'Must contain uppercase')
    .regex(/[0-9]/, 'Must contain a number'),
});

function LoginForm() {
  const { register, handleSubmit, formState: { errors } } = useForm({
    resolver: zodResolver(loginSchema)
  });

  const onSubmit = (data) => {
    // data is already validated and typed before reaching fetch()
    fetch('/api/auth/login', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data)
    });
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      <input {...register('email')} />
      {errors.email && <p>{errors.email.message}</p>}

      <input type="password" {...register('password')} />
      {errors.password && <p>{errors.password.message}</p>}

      <button type="submit">Login</button>
    </form>
  );
}
```

## 1.3 Secure Token Storage

### What is it?
Where you store JWT or session tokens in the browser determines whether XSS can steal them.

### How to secure

```
localStorage       ❌  JavaScript can read it — XSS steals your token trivially
sessionStorage     ❌  Same problem as localStorage
In-memory (RAM)    ✅  XSS can't persist it, but lost on page refresh
HttpOnly Cookie    ✅✅ Browser sends it automatically, JS cannot read it at all
```

**Pattern: Access token in memory + Refresh token in HttpOnly cookie:**
```jsx
// authStore.js — Zustand store, token lives only in memory
import { create } from 'zustand';

const useAuthStore = create((set) => ({
  accessToken: null,

  setAccessToken: (token) => set({ accessToken: token }),

  clearAuth: () => set({ accessToken: null }),
}));

// On app load — silently refresh to get a new access token
// The refresh token is in HttpOnly cookie, sent automatically
async function initAuth() {
  const res = await fetch('/api/auth/refresh', {
    method: 'POST',
    credentials: 'include'  // sends HttpOnly cookie
  });
  if (res.ok) {
    const { accessToken } = await res.json();
    useAuthStore.getState().setAccessToken(accessToken);
  }
}
```

## 1.4 Route Protection (Auth Guards)

### What is it?
Preventing unauthenticated or unauthorized users from accessing protected pages in the React app.

### How to secure

```jsx
// ProtectedRoute.jsx
import { Navigate, Outlet } from 'react-router-dom';
import { useAuthStore } from './authStore';

function ProtectedRoute({ requiredRole }) {
  const { accessToken, user } = useAuthStore();

  // Not logged in
  if (!accessToken) return <Navigate to="/login" replace />;

  // Logged in but wrong role
  if (requiredRole && user?.role !== requiredRole) {
    return <Navigate to="/unauthorized" replace />;
  }

  return <Outlet />;
}

// App.jsx — using the guard
<Routes>
  <Route path="/login" element={<LoginPage />} />

  {/* Protected routes */}
  <Route element={<ProtectedRoute />}>
    <Route path="/dashboard" element={<Dashboard />} />
    <Route path="/profile" element={<Profile />} />
  </Route>

  {/* Admin only */}
  <Route element={<ProtectedRoute requiredRole="admin" />}>
    <Route path="/admin" element={<AdminPanel />} />
  </Route>
</Routes>
```

> ⚠️ This only hides UI. The actual data must be protected in Layer 3 — backend API.

### What is it?
Anything prefixed with `NEXT_PUBLIC_` (Next.js) or `REACT_APP_` (CRA) gets bundled into the JavaScript sent to the browser — anyone can see it.

### How to secure

# LAYER 2 — Middle Layer (API Gateway / Reverse Proxy / CDN)

> This layer sits between the internet and your Node server.  
> It's your **bouncer** — filters, rate-limits, and terminates bad traffic before it ever hits your app code.

---

## 2.1 Reverse Proxy (Nginx)

### What is it?
Nginx sits in front of your Node.js server. The internet talks to Nginx, Nginx talks to Node. Node is never directly exposed.

### How to secure

```nginx
# /etc/nginx/sites-available/myapp

server {
    listen 80;
    server_name yourapp.com;

    # Redirect all HTTP to HTTPS
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name yourapp.com;

    # SSL certificates (via Certbot / Let's Encrypt)
    ssl_certificate     /etc/letsencrypt/live/yourapp.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourapp.com/privkey.pem;

    # Strong TLS only — no TLS 1.0 or 1.1
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # Security headers
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload";
    add_header X-Frame-Options "DENY";
    add_header X-Content-Type-Options "nosniff";
    add_header Referrer-Policy "strict-origin-when-cross-origin";
    add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'";

    # Hide Nginx version from attackers
    server_tokens off;

    # Limit request body size — prevent large payload attacks
    client_max_body_size 10M;

    # Proxy to Node.js
    location /api/ {
        proxy_pass http://localhost:4000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Serve React static build
    location / {
        root /var/www/myapp/build;
        try_files $uri /index.html;
    }
}
```

## 2.2 Rate Limiting at Nginx Level

### What is it?
Throttle requests before they even reach Node — stops DDoS, brute-force, scraping.

```nginx
# Define rate limit zones
http {
    # General API: 30 requests/min per IP
    limit_req_zone $binary_remote_addr zone=api:10m rate=30r/m;

    # Auth endpoints: 5 requests/min per IP (strict)
    limit_req_zone $binary_remote_addr zone=auth:10m rate=5r/m;

    server {
        location /api/ {
            limit_req zone=api burst=10 nodelay;
            limit_req_status 429;
            proxy_pass http://localhost:4000;
        }

        location /api/auth/ {
            limit_req zone=auth burst=3 nodelay;
            limit_req_status 429;
            proxy_pass http://localhost:4000;
        }
    }
}
```

## 2.3 CDN Security (Cloudflare)

### What is it?
A CDN like Cloudflare sits globally in front of your infrastructure — absorbing DDoS attacks, providing WAF (Web Application Firewall), and hiding your origin server's IP.

### How to secure

**Cloudflare WAF rules (UI or API):**
```
Rule 1: Block requests with SQL injection patterns
  → If request contains: ' OR 1=1, UNION SELECT, DROP TABLE
  → Action: Block

Rule 2: Block requests with XSS payloads
  → If request contains: <script>, javascript:, onerror=
  → Action: Block

Rule 3: Geo-blocking
  → If country is in [blocked list]
  → Action: Challenge (CAPTCHA) or Block

Rule 4: Bot protection
  → If user-agent matches known scrapers/scanners
  → Action: Block
```

**Cloudflare DDoS protection:**
```
Under Attack Mode → ON during active attack
  → Forces CAPTCHA challenge for all visitors
  → Absorbs volumetric attacks at edge

Rate Limiting → Block IPs exceeding 100 req/min
```

**Hide origin IP — crucial:**
```bash
# In Cloudflare DNS: all A records set to "Proxied" (orange cloud)
# Your real server IP is hidden behind Cloudflare's IPs
# Attackers can't bypass your WAF by hitting origin directly
```

## 2.4 CORS Configuration

### What is it?
CORS (Cross-Origin Resource Sharing) controls which domains are allowed to call your API from a browser. Without proper CORS, any website could call your API using a logged-in user's cookies.

### How to secure

```js
// In your Express API (Layer 3) but configured at gateway level conceptually

const cors = require('cors');

const allowedOrigins = [
  'https://yourapp.com',
  'https://www.yourapp.com',
  // Never: '*' with credentials: true — that's a security hole
];

app.use(cors({
  origin: (origin, callback) => {
    // Allow server-to-server (no origin header) or whitelisted origins
    if (!origin || allowedOrigins.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error(`CORS blocked: ${origin} not allowed`));
    }
  },
  credentials: true,         // Allow cookies to be sent cross-origin
  methods: ['GET', 'POST', 'PUT', 'DELETE', 'PATCH'],
  allowedHeaders: ['Content-Type', 'Authorization', 'X-CSRF-Token'],
}));
```

## 2.5 SSL/TLS — HTTPS Everywhere

### What is it?
Encrypts all data in transit. Without HTTPS, anyone on the same network (coffee shop WiFi) can read your API requests and steal tokens.

### How to secure

**Auto-renewing SSL with Certbot:**
```bash
# Install Certbot
sudo apt install certbot python3-certbot-nginx

# Get certificate
sudo certbot --nginx -d yourapp.com -d www.yourapp.com

# Auto-renewal (certbot installs a cron job automatically)
sudo certbot renew --dry-run  # test it
```

**Force HTTPS in Node (fallback):**
```js
// Redirect HTTP to HTTPS if not behind Nginx/proxy
app.use((req, res, next) => {
  if (req.headers['x-forwarded-proto'] !== 'https' && process.env.NODE_ENV === 'production') {
    return res.redirect(301, `https://${req.hostname}${req.originalUrl}`);
  }
  next();
});
```

# LAYER 3 — Backend (Node.js / Express)

> This is the **source of truth**. Every security check here is mandatory.  
> Never trust anything from Layer 1 or Layer 2 — validate and enforce everything here.

---

## 3.1 Authentication — JWT + HttpOnly Cookies

### What is it?
Verifying who the user is on every single protected request.

### How to secure

**Login — issue tokens:**
```js
const jwt = require('jsonwebtoken');
const bcrypt = require('bcrypt');

app.post('/api/auth/login', authLimiter, async (req, res) => {
  const { email, password } = req.body;

  // 1. Find user
  const user = await db.query('SELECT * FROM users WHERE email = $1', [email]);
  if (!user.rows[0]) {
    // Generic message — don't reveal if email exists
    return res.status(401).json({ error: 'Invalid credentials' });
  }

  // 2. Compare password hash
  const valid = await bcrypt.compare(password, user.rows[0].password_hash);
  if (!valid) return res.status(401).json({ error: 'Invalid credentials' });

  // 3. Issue short-lived access token
  const accessToken = jwt.sign(
    { userId: user.rows[0].id, role: user.rows[0].role },
    process.env.JWT_SECRET,
    { expiresIn: '15m' }
  );

  // 4. Issue refresh token
  const refreshToken = jwt.sign(
    { userId: user.rows[0].id },
    process.env.JWT_REFRESH_SECRET,
    { expiresIn: '7d' }
  );

  // 5. Store refresh token hash in DB (so we can revoke it)
  await db.query(
    'INSERT INTO refresh_tokens (user_id, token_hash) VALUES ($1, $2)',
    [user.rows[0].id, await bcrypt.hash(refreshToken, 10)]
  );

  // 6. Send refresh token in HttpOnly cookie
  res.cookie('refreshToken', refreshToken, {
    httpOnly: true,          // JS cannot access
    secure: true,            // HTTPS only
    sameSite: 'strict',      // CSRF protection
    maxAge: 7 * 24 * 60 * 60 * 1000  // 7 days
  });

  // 7. Send access token in response body (stored in memory on client)
  res.json({ accessToken });
});
```

**Auth middleware — protect every route:**
```js
// middleware/auth.js
const jwt = require('jsonwebtoken');

function authenticate(req, res, next) {
  const authHeader = req.headers.authorization;

  if (!authHeader?.startsWith('Bearer ')) {
    return res.status(401).json({ error: 'No token provided' });
  }

  const token = authHeader.split(' ')[1];

  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded;    // { userId, role, iat, exp }
    next();
  } catch (err) {
    if (err.name === 'TokenExpiredError') {
      return res.status(401).json({ error: 'Token expired', code: 'TOKEN_EXPIRED' });
    }
    return res.status(403).json({ error: 'Invalid token' });
  }
}

module.exports = authenticate;
```

## 3.2 Authorization — Role-Based Access Control (RBAC)

### What is it?
Authentication = who you are. Authorization = what you're allowed to do.  
This is the most commonly missed layer — developers add auth but forget authz.

### How to secure

```js
// middleware/authorize.js
function authorize(...roles) {
  return (req, res, next) => {
    // req.user is set by authenticate() middleware
    if (!req.user) return res.status(401).json({ error: 'Unauthorized' });

    if (!roles.includes(req.user.role)) {
      return res.status(403).json({ error: 'Forbidden — insufficient permissions' });
    }

    next();
  };
}

// Usage
app.get('/api/users',
  authenticate,
  authorize('admin'),           // only admins
  getAllUsersHandler
);

app.put('/api/posts/:id',
  authenticate,
  authorize('admin', 'editor'),  // admins and editors
  updatePostHandler
);

app.get('/api/profile',
  authenticate,                  // any authenticated user
  getProfileHandler
);
```

**Resource-level authorization (prevent IDOR):**
```js
// IDOR = Insecure Direct Object Reference
// User A accessing User B's data by changing the ID in the URL

app.get('/api/orders/:orderId', authenticate, async (req, res) => {
  const order = await db.query(
    'SELECT * FROM orders WHERE id = $1',
    [req.params.orderId]
  );

  if (!order.rows[0]) return res.status(404).json({ error: 'Not found' });

  // ✅ Check ownership — does this order belong to the requesting user?
  if (order.rows[0].user_id !== req.user.userId && req.user.role !== 'admin') {
    return res.status(403).json({ error: 'Forbidden' });
    // Don't reveal the order exists — 404 is safer in some cases
  }

  res.json(order.rows[0]);
});
```

## 3.3 Input Validation & Sanitization

### What is it?
Every piece of data coming from the client — body, params, query — must be validated and sanitized before touching your database or business logic.

### How to secure

**express-validator:**
```js
const { body, param, query, validationResult } = require('express-validator');

// Validation middleware
const validateCreatePost = [
  body('title')
    .trim()
    .notEmpty().withMessage('Title is required')
    .isLength({ max: 200 }).withMessage('Title too long')
    .escape(),                              // converts < > & " ' to HTML entities

  body('content')
    .trim()
    .notEmpty().withMessage('Content is required')
    .isLength({ max: 10000 }),
    // Don't escape content if you allow rich text — sanitize differently

  body('tags')
    .isArray({ max: 10 }).withMessage('Max 10 tags')
    .optional(),

  param('id')
    .isInt({ min: 1 }).withMessage('Invalid ID'),

  // Process validation results
  (req, res, next) => {
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({
        error: 'Validation failed',
        details: errors.array()
      });
    }
    next();
  }
];

app.post('/api/posts',
  authenticate,
  validateCreatePost,
  createPostHandler
);
```

**Always use parameterized queries — never string concatenation:**
```js
const { Pool } = require('pg');
const pool = new Pool();

// ❌ SQL Injection — NEVER do this
async function getUserBad(email) {
  const query = `SELECT * FROM users WHERE email = '${email}'`;
  // Attacker passes: ' OR '1'='1 — dumps entire table
  return pool.query(query);
}

// ✅ Parameterized — safe always
async function getUserSafe(email) {
  return pool.query('SELECT * FROM users WHERE email = $1', [email]);
}

// ✅ With pg — multiple params
async function createUser(name, email, passwordHash) {
  return pool.query(
    'INSERT INTO users (name, email, password_hash) VALUES ($1, $2, $3) RETURNING id',
    [name, email, passwordHash]
  );
}
```

## 3.4 Password Security

```js
const bcrypt = require('bcrypt');

const SALT_ROUNDS = 12;  // Higher = slower = harder to brute-force

// Hash on registration
async function hashPassword(plaintext) {
  return bcrypt.hash(plaintext, SALT_ROUNDS);
}

// Verify on login
async function verifyPassword(plaintext, hash) {
  return bcrypt.compare(plaintext, hash);
  // bcrypt.compare is timing-safe — prevents timing attacks
}

// Usage
app.post('/api/auth/register', async (req, res) => {
  const { email, password } = req.body;

  // Check password strength on backend too
  if (password.length < 8) {
    return res.status(400).json({ error: 'Password too short' });
  }

  const hash = await hashPassword(password);

  await db.query(
    'INSERT INTO users (email, password_hash) VALUES ($1, $2)',
    [email, hash]
  );

  res.status(201).json({ message: 'User created' });
});
```

## 3.5 Rate Limiting (Express Level)

```js
const rateLimit = require('express-rate-limit');
const RedisStore = require('rate-limit-redis');
const redis = require('redis');

const redisClient = redis.createClient({ url: process.env.REDIS_URL });

// General API limiter
const apiLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,  // 15 minutes
  max: 100,
  standardHeaders: true,
  legacyHeaders: false,
  store: new RedisStore({ client: redisClient }),  // shared across instances
  message: { error: 'Too many requests. Try again in 15 minutes.' }
});

// Strict limiter for auth routes
const authLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 5,
  skipSuccessfulRequests: true,  // only counts failures
  store: new RedisStore({ client: redisClient }),
  message: { error: 'Too many failed attempts. Try again later.' }
});

app.use('/api/', apiLimiter);
app.use('/api/auth/login', authLimiter);
app.use('/api/auth/register', authLimiter);
```

## 3.7 Secrets Management

## 3.8 Error Handling — Don't Leak Stack Traces

```js
// ❌ Default Express error — leaks internal details
// Error: Cannot read property 'id' of undefined
//   at /app/src/handlers/user.js:42:18
//   at Layer.handle [as handle_request] ...

// ✅ Global error handler — generic messages in production
app.use((err, req, res, next) => {
  // Log full error internally
  console.error({
    message: err.message,
    stack: err.stack,
    path: req.path,
    method: req.method,
    userId: req.user?.userId,
    timestamp: new Date().toISOString()
  });

  // Send generic message to client
  const statusCode = err.statusCode || 500;
  const message = process.env.NODE_ENV === 'production'
    ? 'Something went wrong'   // hide details in production
    : err.message;             // show details in development

  res.status(statusCode).json({ error: message });
});

// Custom error class
class AppError extends Error {
  constructor(message, statusCode) {
    super(message);
    this.statusCode = statusCode;
    this.isOperational = true;
  }
}

// Usage in handlers
throw new AppError('User not found', 404);
throw new AppError('Forbidden', 403);
```

## 3.9 CSRF Protection

```js
const csrf = require('csurf');

// Using cookie-based CSRF
const csrfProtection = csrf({
  cookie: {
    httpOnly: false,     // Frontend JS needs to read this one token
    secure: true,
    sameSite: 'strict'
  }
});

// Send token to frontend
app.get('/api/csrf-token', csrfProtection, (req, res) => {
  res.json({ csrfToken: req.csrfToken() });
});

// Protect all state-changing routes
app.use('/api/', csrfProtection);
```

# Interview Summary

| Layer | Responsibility | Key Tools |
|---|---|---|
| **Frontend** | Reduce attack surface, UX validation, safe rendering, no secrets in bundle | React JSX, DOMPurify, Zod, react-router guards |
| **Middle** | Traffic filtering, TLS, rate limiting, WAF, CORS, DDoS protection | Nginx, Cloudflare, Certbot |
| **Backend** | Enforce all auth/authz, validate all input, parameterized queries, secure tokens, safe errors | Helmet, bcrypt, JWT, express-validator, express-rate-limit, pg |

> *"Security is a layered contract — the frontend reduces surface area and gives UX feedback, the middle layer filters malicious traffic and enforces HTTPS, and the backend is the enforcer that trusts nothing and validates everything. No single layer is enough alone."*
